feat: Production-grade codebase indexing with semantic embeddings and sqlite-vector integration

This commit implements a complete production-grade codebase indexing system with
semantic embedding capabilities, real-time file watching, and native vector search
via sqlite-vector extension.

## Phase 1-2: IPC Architecture & File Tree Indexing
- Moved all SQLite operations to extension host (Node process)
- Implemented IPC/RPC protocol between renderer and ext-host
- Added file tree indexing with workspace root detection
- Implemented SQLite persistence with schema versioning
- Added deterministic migrations and restart-safe state

## Phase 3: Incremental Chunk Indexing
- Implemented content-based chunking (200 lines, 20 line overlap)
- Added hash-based invalidation (sha256) for changed files
- Created chunks table with stable chunkId generation
- Implemented incremental saved-file indexing
- Added file status tracking (discovered/indexed/error)

## Phase 4: Real-time & Background Indexing
- Implemented file system watchers for create/change/delete events
- Added debouncing and batching for rapid file changes
- Implemented background indexing for discovered files
- Added pause/resume functionality with accumulated change tracking
- Ensured restart safety and watcher reattachment

## Phase 5-6: Semantic Embeddings & Local ONNX Runtime
- Implemented LocalOnnxEmbeddingRuntime with ONNX model support
- Added ModelManager for automatic model download and installation
- Created wrapper runtime with ONNX-first, hash-fallback strategy
- Implemented embedding queue processing with batching
- Added embedding progress tracking and status reporting

## Technical Details
- Extension host (Node) handles all SQLite and file system operations
- Renderer communicates via IPC/RPC protocol
- Embeddings stored as Float32Array BLOB for sqlite-vector compatibility
- Vector queries use scalar functions: vector_cosine_distance(vector, query)
- Platform-aware sqlite-vector binary download (macOS/Linux/Windows, arm64/x64)
- Schema versioning ensures safe migrations
- Content hashing (sha256) for efficient change detection
